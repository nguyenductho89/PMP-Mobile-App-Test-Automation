## MỤC ĐÍCH VÀ TÁC DỤNG

**Mục đích:** Issue Log theo dõi tất cả các vấn đề (issues) phát sinh trong dự án và trạng thái giải quyết để đảm bảo issues được giải quyết kịp thời.

**Tác dụng:**
- Đảm bảo issues được giải quyết kịp thời
- Tránh issues bị bỏ sót hoặc quên
- Hỗ trợ escalation khi cần
- Track issue resolution time
- Identify patterns và trends
- Improve issue management

**Đối tượng sử dụng:**
- **Project Manager:** Issue management
- **Team Members:** Report và track issues
- **Stakeholders:** Understand project issues

**Thời điểm sử dụng:**
- Tạo trong Planning phase
- Cập nhật khi có issues mới
- Review trong status meetings

**Mối liên hệ với các tài liệu khác:**
- Liên kết với Risk Register
- Tích hợp với Change Management Plan
- Hỗ trợ Communications Management Plan

## 5. ISSUE STATUS

| Status | Description |
|--------|-------------|
| **Open** | Issue reported, not yet resolved |
| **In Progress** | Issue being worked on |
| **Resolved** | Issue resolved, pending verification |
| **Closed** | Issue verified và closed |
| **Deferred** | Issue deferred to later |
| **Cancelled** | Issue no longer applicable |

---

## 6. ISSUE RESOLUTION PROCESS

1. **Report:** Issue reported và logged
2. **Assess:** Assess priority và impact
3. **Assign:** Assign owner
4. **Resolve:** Work on resolution
5. **Verify:** Verify resolution
6. **Close:** Close issue
