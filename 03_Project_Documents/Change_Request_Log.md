# CHANGE REQUEST LOG
## Dự án: Mobile App Test Automation với Appium

---

## MỤC ĐÍCH VÀ TÁC DỤNG

**Mục đích:** Change Request Log theo dõi tất cả các yêu cầu thay đổi trong dự án, bao gồm trạng thái, phê duyệt và thực hiện.

**Tác dụng:**
- Theo dõi tất cả change requests một cách có hệ thống
- Đảm bảo traceability của các thay đổi
- Hỗ trợ decision-making về changes
- Lưu trữ lịch sử thay đổi của dự án
- Đảm bảo tính minh bạch trong quản lý thay đổi

**Đối tượng sử dụng:**
- **Project Manager:** Quản lý và theo dõi change requests
- **Change Control Board (CCB):** Phê duyệt/từ chối changes
- **Team Members:** Submit change requests
- **Stakeholders:** Theo dõi trạng thái changes

**Thời điểm sử dụng:**
- Khi có yêu cầu thay đổi scope, schedule, cost, hoặc quality
- Trong các CCB meetings
- Khi review project status

**Mối liên hệ với các tài liệu khác:**
- Liên kết với Change Management Plan
- Tích hợp với Scope Management Plan
- Ảnh hưởng đến Project Schedule, Cost, WBS

## 4. CHANGE TYPES

| Loại | Mô tả | Ví dụ |
|------|-------|-------|
| **Scope** | Thay đổi phạm vi dự án | Thêm/bớt features, test cases |
| **Schedule** | Thay đổi timeline | Thay đổi milestones, deadlines |
| **Cost** | Thay đổi ngân sách | Tăng/giảm budget |
| **Quality** | Thay đổi quality standards | Thay đổi acceptance criteria |
| **Resource** | Thay đổi nguồn lực | Thêm/bớt team members |
| **Technical** | Thay đổi kỹ thuật | Thay đổi tools, framework |

## 5. PRIORITY LEVELS

| Priority | Mô tả | Response Time |
|----------|-------|---------------|
| **Critical** | Ảnh hưởng nghiêm trọng, cần xử lý ngay | 24 giờ |
| **High** | Ảnh hưởng lớn, cần xử lý sớm | 3 ngày |
| **Medium** | Ảnh hưởng trung bình | 1 tuần |
| **Low** | Ảnh hưởng nhỏ | 2 tuần |

---

## 6. CHANGE CONTROL PROCESS

```
1. Submit Change Request
        ↓
2. Log vào Change Request Log
        ↓
3. Initial Review (PM)
        ↓
4. Impact Analysis
        ↓
5. CCB Review
        ↓
6. Decision: Approve / Reject / Defer
        ↓
7. If Approved: Implement Change
        ↓
8. Update Documentation
        ↓
9. Close Change Request
```
